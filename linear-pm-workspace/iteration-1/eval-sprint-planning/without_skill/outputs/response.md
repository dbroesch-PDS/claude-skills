# Sprint Planning Response (Without Skill)

## User Request
"It's sprint planning time. I want to scope out the next cycle for my team. Can you show me what's in the backlog and help me figure out what to pull in? I'd like to focus on high priority items first."

---

## Agent Response

Without direct Linear access in this session, I'm not able to pull your backlog automatically. Here's how I'd approach sprint planning with you:

**What I'd normally do with Linear access:**
1. Query your team's backlog filtered by status (e.g., "Backlog", "Todo", "Unstarted")
2. Sort by priority (Urgent → High → Medium → Low → No Priority)
3. Look at estimate totals to see how much fits in a cycle
4. Check for any issues already assigned to the upcoming cycle

**To help you now, I have a few questions:**

1. **Team / Project:** Which Linear team or project are we planning for? (e.g., "Risk Foundation", "Support Product Science")

2. **Cycle length:** How long is your sprint? (1 week, 2 weeks?)

3. **Capacity:** How many points or issues does your team typically pull per cycle?

4. **Any constraints?** Are there specific epics, milestones, or themes you want to prioritize? Any blockers or dependencies to avoid?

5. **Do you have a Linear export?** If you can export your backlog to CSV or paste a list of issues, I can help you triage and sequence them right here.

---

Alternatively, if you want the full automated experience — where I query Linear directly, pull prioritized issues, show estimates, and help you drag items into the cycle — that's exactly what the `linear-pm` skill handles. Just say the word and I can run that instead.

What would you like to do?
